# Мобильные вкладки CRM: требования Bitrix24

Официальная документация Bitrix24 по методу [`placement.bind`](https://apidocs.bitrix24.ru/api-reference/placement/placement-bind.html) и разделу «[Размещения в мобильном приложении](https://training.bitrix24.com/rest_help/mobile/mobileapp/placements.php)» подчёркивает три момента, без которых кастомная вкладка не появится в карточке сделки в мобильном приложении:

1. **Локализации через `LANG_ALL`.** Для каждой доступной локали нужно указать название вкладки и описание. Минимальный набор — `ru` и `en`. Без `LANG_ALL` мобильный клиент игнорирует вкладку.
2. **Опция `support_mobile`.** В объекте `OPTIONS` обязательна установка `support_mobile = 'Y'` (мобильный клиент до 2022 года принимает и верблюжий регистр `supportMobile`). Одновременно полезно оставлять `register = 'Y'`, чтобы при переустановке приложения таб не терялся.
3. **HTTP POST для вложенных параметров.** При передаче массивов (`LANG_ALL`, `OPTIONS`) метод нужно вызывать через POST: Bitrix24 ожидает массив параметров в теле, запросы через GET с длинной строкой могут silently игнорироваться.

Эти требования учтены в `public/bitrix/install.php`: теперь переустановка приложения повторно привязывает вкладку «Рейсы» к `CRM_DEAL_DETAIL_TAB` с поддержкой мобильного приложения.
